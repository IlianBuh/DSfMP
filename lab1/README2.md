Цель лабораторных работ: разработать мобильное приложение с поэтапным внедрением функционала. 
Технология разработки на выбор: React Native. Для тестирования установлен Android Studio.  
Тема приложения: конструктор и хранилище профессиональных резюме
Требования: 
1. Пользовательский интерфейс: минимум 3 страницы (например, основной экран, экран просмотра деталей, экран настроек). 
2. Экран загрузки: отображение сплеш-экрана при запуске приложения. 
3. Поддержка локализации: минимум два языка (например, русский и английский). 
4. Тема интерфейса: выбор светлой или темной темы с сохранением после перезагрузки. 
5. Локальная база данных: создание, удаление и редактирование записей с минимум тремя полями (например, заголовок, описание, дата). 

6. Архитектура: разделение бизнес-логики и уровня представления 
(MVVM, MVP или др). 
7. Веб-API: получение данных из внешнего сервиса (NewsAPI, 
OpenWeatherMap и др). 
8. Мониторинг: проверка наличия интернет-соединения. 
9. Оффлайн-режим: кэширование данных из API (использовать локальную 
базу данных или кэш-библиотеки).


10. Инструменты работы с данными: поиск, фильтрация, сортировка и 
алгоритм нечеткого поиска (по ключевым словам, датам или 
категориям). 
11. Настройка уведомлений (по расписанию). 
12. Удаленная база данных: сохранение записей (PostgreSQL) и изображений (ImageKit).

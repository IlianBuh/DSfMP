# Отчет по проекту "Resume Builder"

## 1. Используемые технологии и инструменты

### Платформа разработки
В качестве основной платформы разработки выбран фреймворк **React Native** совместно с набором инструментов **Expo** и языком программирования **TypeScript**. 

**Причины выбора и преимущества:**
*   **Кроссплатформенность:** Одна и та же кодовая база компилируется в нативные приложения для iOS и Android, что существенно сокращает время и стоимость разработки.
*   **JavaScript/TypeScript экосистема:** Использование TypeScript предоставляет строгую типизацию, автодополнение и раннее обнаружение ошибок. Экосистема JS позволяет переиспользовать множество готовых библиотек.
*   **Expo:** Expo берет на себя сложную настройку нативных проектов, предоставляет готовую инфраструктуру для сборки, тестирования (Expo Go) и содержит набор оптимизированных модулей (например, `expo-sqlite`, `expo-splash-screen`).
*   **Преимущества перед конкурентами (например, Flutter или Swift/Kotlin):** В отличие от нативной разработки (Swift/Kotlin), не нужно писать два разных приложения. В отличие от Flutter (Dart), React Native использует привычный многим веб-разработчикам React API (JSX, хуки) и позволяет легче внедрять проект в существующую инфраструктуру с веб-версиями.
*   **Hot Reloading:** Мгновенное обновление интерфейса при изменении кода без необходимости полной пересборки (Fast Refresh).

### Как происходит навигация
Для маршрутизации в проекте используется библиотека **React Navigation** (версии 7). 
Архитектура навигации построена по комбинированному принципу:
*   **Bottom Tabs Navigator** (`@react-navigation/bottom-tabs`): Основное меню для переключения между главными вкладками приложения (например, список резюме и настройки).
*   **Native Stack Navigator** (`@react-navigation/native-stack`): Обеспечивает вложенную навигацию внутри вкладки списков. При переходе на создание или редактирование конкретного резюме происходит погружение в стек экранов с возможностью возврата (анимация свайпа, кнопка "Назад"). Навигация тесно интегрирована с системными темами приложения интерфейсов (провайдер `NavigationContainer` инициализируется вместе с глобальной темой приложения).

### Работа с СУБД
Для хранения данных используется **SQLite** — локальная легковесная реляционная база данных.
Интеграция реализована через библиотеку `expo-sqlite`. 
*   **Инициализация:** При старте приложения выполняется асинхронное подключение к файлу `resumes.db` и создание таблиц (через `CREATE TABLE IF NOT EXISTS`).
*   **Доступ к данным:** Логика БД вынесена в отдельный слой (DAO), где реализованы CRUD-операции. Запросы в БД (`INSERT`, `SELECT`, `UPDATE`, `DELETE`) выполняются асинхронно через методы драйвера вроде `runAsync` или `getAllAsync` с использованием параметризованных запросов во избежание SQL-инъекций. 
*   Таблица связывает резюме пользователя по полям `fullName`, `profession`, `email`, `phone`, `experience`, `education`, `skills` и `createdAt`.

## 2. Основные особенности проекта и функциональность

Проект "Resume Builder" представляет собой приложение для локального создания и управления портфолио/резюме на мобильном устройстве.

**Основная функциональность:**
*   **Управление списком (CRUD):** 
    *   Создание нового резюме с помощью подробной формы (Имя, профессия, контакты, опыт, образование, навыки). Валидация обязательных полей.
    *   Отображение списка всех созданных резюме на главной странице с сортировкой по дате создания.
    *   Редактирование ранее добавленных резюме (форма заполняется существующими данными из БД).
    *   Удаление резюме.
*   **Мультиязычность (i18n):** В проекте внедрена библиотека `i18next` и `react-i18next` для динамического переключения языков интерфейса приложения "на лету" без перезагрузки среды.
*   **Темы оформления:** Разработан встроенный провайдер тем (`ThemeContext`), который поддерживает переключение светлой и темной темы (Dark Mode), затрагивая сразу все элементы UI навигации, кнопок и полей ввода. Определение текущей темы может происходить из системных настроек.
*   **Стартовый экран (Splash Screen):** Программное удержание сплеш-экрана (`expo-splash-screen`) до момента, пока база данных не инициализируется и не загрузятся необходимые пресеты. Приложение отображается пользователю только тогда, когда оно полностью готово к работе.

## 3. Теоретическая основа разработки 

### Теоретическая база разработки мобильного приложения (React Native)
Разработка на React Native опирается на принципы декларативного пользовательского интерфейса:
1.  **Компонентный подход:** Интерфейс строится из изолированных, переиспользуемых модулей (компонентов).
2.  **Жизненный цикл и хуки (Hooks):** Использование `useState` для создания локальных переменных состояния формы, `useEffect` для подписки на внешние ресурсы (например, инициализации компонента, обращения к СУБД после монтирования) и `useContext` для внедрения глобальных зависимостей (язык, тема).
3.  **Стейт-менеджмент:** Архитектура управления состоянием. Разделение бизнес-логики (запросы в базу) и представления (JSX).
4.  **Асинхронность (Promise, Async/Await):** Поскольку JS выполняется в одном потоке, ресурсоемкие задачи, такие как чтение с диска SQLite и рендеринг тяжелых списков, осуществляются асинхронно или делегируются нативным модулям мобильной ОС, чтобы не блокировать интерфейс React (UI Thread).

### Теоретическая основа проекта
Проект построен по слоистой архитектуре *(Layered Architecture)*, разделяющий зоны ответственности:
1.  **Слой представления (UI Layer - экраны):** Папка `src/screens`. Файлы `HomeScreen`, `ResumeScreen`, `SettingsScreen`. Отвечают исключительно за отрисовку и взаимодействие с пользователем (собирают ввод из TextInput, передают объект дальше, обрабатывают нажатия). 
2.  **Слой навигации (Routing Layer):** Папка `src/navigation`. Управляет переходами и стеком истории интерфейса.
3.  **Слой доступа к данным (Database / DAO Layer):** Папка `src/database`. Модуль `db.ts` абстрагирует в себе работу с SQL-сущностями. Экраны обращаются только к функциям (например, `getResumes()`, `updateResume()`), ничего не зная о самом SQL-синтаксисе.
4.  **Инфраструктурный слой (Context & i18n):** Глобальные провайдеры для темы и локализации, оборачивающие основу приложения в `App.tsx`. Обеспечивают единое окружение для всех страниц приложения в стеке навигатора.

## 4. Выводы по проекту

Приложение "Resume Builder" представляет собой полноценный рабочий MVP, внедряющий ключевые аспекты нативной мобильной разработки: работу с локальными реляционными базами данных (SQLite), сложную стек-табовую навигацию (React Navigation), обработку пользовательского ввода с виртуальной клавиатурой, а также управление глобальными контекстами (Dark Theme / i18n). 

Выбор React Native и Expo позволил создать оптимизированное приложение, которое сразу адаптировано как под iOS, так и Android с единой монолитной кодовой базой компонентов, при этом предоставляя нативный пользовательский опыт (нативные стеки экранов, нативные шрифты и анимации навигации). Благодаря четкому разделению интерфейса, навигации и логики работы СУБД архитектура проекта остается масштабируемой и удобной для добавления нового функционала (например, экспорта резюме в PDF или синхронизации с облаком).
